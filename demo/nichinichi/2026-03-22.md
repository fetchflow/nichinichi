13:00 | decided to scope rate limiting to per-user per-endpoint, not global bucket @acme #decision

       Options:
       - Global rate limit: simple, but one noisy user can degrade everyone
       - Per-user global: better isolation, misses per-endpoint abuse
       - Per-user per-endpoint: most precise, slightly more Redis key overhead
       Went with per-user per-endpoint. Redis key pattern: ratelimit:{user_id}:{endpoint}.
       TTL = window size (60s for most, 3600s for auth endpoints).
